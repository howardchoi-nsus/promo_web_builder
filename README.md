# promo_web_builder
2026.06.25

**목적**
본 프로젝트는 기존 WordPress 기반 GG Promo Builder의 한계를 보완하고, 향후 WordPress 개선, 내부 CMS 확장, 신규 빌더, AI 기반 빌더 전환 가능성을 검증하기 위한 애자일 POC이다.
1차 목표는 완성형 플랫폼 구축이 아니라, 사용자의 프로모션 요구사항을 입력받아 검토 가능한 프로모션 페이지 초안을 생성하는 E2E 흐름을 만드는 것이다.
핵심 방향은 다음과 같다.
NLP가 요구사항을 해석하고, Rule Base가 운영 정책과 필수 구조를 통제하며, Template Family와 Component를 조합해 검수 가능한 프로모션 페이지 초안을 생성한다.

**배경**
현재 프로모션 페이지는 WordPress 기반 빌더로 운영 중이며, 각 서비스 리전의 프로모션 담당자가 관리자 페이지에서 템플릿을 선택하고, 템플릿별 입력 항목을 채워 랜딩 페이지를 생성한다.
현재 운영상의 주요 이슈는 다음과 같다.
템플릿마다 입력 항목이 달라 사용 경험이 일관되지 않음
일반적인 디자인 템플릿은 많이 사용되지만 디자인 만족도가 낮음
다이나믹한 디자인 템플릿은 관리 통제가 어려움
WordPress 기반 유지 여부는 확정되지 않음
내부 CMS에 프로모션 빌더를 추가하는 방안도 고려 중
Affiliate, Bonus Code, Q-TAG, Terms, Responsible Gaming 등 운영 정책 검증이 필요함
따라서 초기에는 특정 플랫폼을 확정하지 않고, 작은 E2E 흐름을 먼저 구현한 뒤 반복적으로 완성도를 높인다.

**기획 원칙**
Phase 방식보다 Agile E2E 방식으로 진행
Template-first가 아니라 Component-first
단일 템플릿 재현이 아니라 Template Family 기반 설계
NLP/AI는 해석과 초안 생성 담당
Rule Base는 필수 구조, 정책, 운영 안전성 통제 담당
AI 생성 결과는 즉시 배포물이 아니라 운영자가 검수하는 초안
플랫폼 결정은 추후 검토
WordPress, 내부 CMS, 신규 빌더 모두 고려 가능한 구조로 설계
