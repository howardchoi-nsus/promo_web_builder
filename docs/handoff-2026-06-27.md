### 1. B섹션 입력 구조 재정리

- 기존 단순 페이지 생성 흐름을 `UI 디자인 이미지 생성` 흐름으로 전환했습니다.
- 현재 사용자 흐름은 아래 구조입니다.

```text
A. 디자인 스타일 콘셉트 선택
  -> 하위 디자인 MD 선택
B. 프로모션 빌더
  -> Temp.4 기준 간편 프로모션 브리프 입력
  -> AI 섹션 초안 생성/수정
  -> 색상 / 폰트 스타일 선택 및 재정의
  -> UI 디자인 생성
C. 결과
  -> 생성된 UI 디자인 결과 확인
```

- B섹션에는 Temp.4 기준 주요 입력 구조를 반영했습니다.
  - Hero Banner
  - Step Bar
  - Content CTA
  - Image Text Row
  - Terms / Footer
- 비전문가도 사용할 수 있도록 `간편 프로모션 브리프 -> AI 섹션 초안 -> UI 디자인 생성` 흐름으로 정리했습니다.
- `Simple` / `Advanced` 모드 값은 유지하되, 화면 표시 문구는 `간편` / `고급`으로 한글화했습니다.

주요 파일:

- `prototype/index.html`
- `prototype/app.js`

### 2. n8n UI 디자인 이미지 생성 workflow 정리

- 기존 페이지 생성 중심 흐름이 아니라 `UI 디자인 이미지 생성`에 맞는 n8n workflow를 기준으로 정리했습니다.
- `n8n/promo-ui-design-image-generator.workflow.json`에서 이미지 생성용 prompt 빌드 단계를 강화했습니다.
- prompt에 포함되는 주요 컨텍스트:
  - 선택된 MD
  - Design Concept
  - Layout Philosophy
  - spacing rhythm
  - composition
  - component style
  - depth/effects
  - shape/radius
  - dos/donts
  - Color / Font
  - Promo 입력값
  - Temp.4 sectionInputs
- 결과는 `designUrl`, `imageUrl`, `layoutMapping`, `mdComplianceMap` 중심으로 확인하는 방향입니다.
- `mdComplianceMap`에는 단순 색상/폰트 반영이 아니라 MD 철학 반영 여부를 증명하도록 필드를 강화했습니다.

주요 파일:

- `n8n/promo-ui-design-image-generator.workflow.json`

## 현재 구현 상태

현재 POC는 아래 흐름까지 동작하는 구조입니다.

```text
디자인 MD 로드
  -> 스타일 콘셉트 그룹 선택
  -> 하위 MD 선택
  -> 프로모션 간편 입력
  -> Temp.4 섹션 초안 생성
  -> 색상/폰트 기준 선택 및 재정의
  -> n8n UI 디자인 이미지 생성 요청
  -> 생성 결과 URL 또는 로컬 payload 미리보기 확인
```

브라우저에서 정적 서버로 QA할 경우 `/api/design-documents`가 없기 때문에 fallback dummy 데이터가 표시됩니다. Vercel/API 환경에서는 Neon Postgres의 디자인 MD 목록을 불러오는 구조입니다.

## 남은 작업

### 1. 실제 n8n 결과 품질 QA

- 현재 prompt는 MD 철학 반영을 강화했지만, 실제 이미지 생성 결과를 여러 브랜드 MD로 테스트해야 합니다.
- 특히 확인할 것:
  - MD의 layoutPhilosophy가 실제 구도에 반영되는지
  - 색상/폰트만 바뀐 일반 랜딩 페이지처럼 나오지 않는지
  - Temp.4 섹션 순서가 지켜지는지
  - 약관/푸터가 누락되지 않는지
  - `mdComplianceMap`이 의미 있는 설명을 반환하는지

### 2. Advanced Mode 고도화

- 현재는 간편 브리프 기반으로 section draft를 만드는 구조입니다.
- `Advanced` 모드는 값만 존재하고, UX 구분은 아직 충분히 명확하지 않습니다.
- 다음 단계에서는 Advanced Mode에서 각 section을 직접 세밀하게 수정할 수 있도록 화면 밀도와 편집 흐름을 정리해야 합니다.

### 3. UI 디자인 이미지 -> 프론트 코드 변환 단계 추가

- 현재는 UI 디자인 이미지 생성 단계입니다.
- 다음 목표는 사용자가 디자인 이미지를 확인/승인한 뒤 HTML/CSS/Vue 코드로 변환하는 흐름입니다.
- n8n workflow도 별도 `Code Generation` 단계가 필요합니다.
- 예상 흐름:

```text
UI 디자인 이미지 생성
  -> 운영자 확인 / 승인
  -> 코드 생성 요청
  -> HTML/CSS/Vue 산출
  -> 미리보기
  -> 운영 검수
```

### 4. DB 저장 구조 검토

운영화하려면 생성 결과 저장용 테이블이 필요합니다.

검토 후보:

- `promo_design_runs`

## 주의 사항

- `.DS_Store`가 수정 상태로 남을 수 있습니다. 코드 변경과 무관한 macOS 메타데이터 파일입니다.
- 정적 서버로 열면 API가 없어서 fallback dummy 데이터를 사용합니다.
- `n8n Webhook URL`, 폰트명, 마켓명, CTA 등은 의도적으로 영문/기술 용어를 유지한 부분이 있습니다.
- 루트 `index.html`, `generated.html`은 실제 앱이 아니라 `prototype` 하위 화면으로 넘기는 redirect shell입니다.
