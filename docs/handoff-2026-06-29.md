# Handoff - 2026-06-29

## 오늘 진행 요약

- Promo UI Design 생성 결과를 임시 상태가 아니라 DB와 Vercel Blob에 저장하는 흐름으로 전환했다.
- n8n에서 생성된 이미지 결과를 `promo-020` 기준으로 저장, 조회, 보기 화면까지 정상 확인했다.
- 기존 n8n view webhook 의존 구조에서 Vercel API 기반 보기 URL로 전환하는 방향을 검증했다.

## 완료된 주요 내용

### 1. 생성 이미지 영속 저장 구조 적용

- 생성된 UI 디자인 이미지를 Vercel Blob에 저장하도록 API를 구성했다.
- 생성 실행 정보와 이미지 메타데이터는 DB 테이블에 저장하도록 구성했다.
- 페이지 새로고침이나 n8n 실행 데이터 만료 후에도 runKey 기준으로 다시 조회할 수 있는 구조가 되었다.

### 2. DB 테이블 구성

- 생성 실행 단위와 이미지 자산을 분리해서 저장하는 테이블을 추가했다.
- 적용 대상 migration:
  - `db/migrations/004_promo_design_image_storage.sql`
- 주요 개념:
  - 디자인 생성 실행 정보
  - 생성 이미지 asset 정보
  - primary image 구분
  - Blob storage key와 원본/프록시 URL 관리

### 3. Vercel API 구성

- 생성 결과 저장 API를 추가했다.
- 생성 디자인 보기 API를 추가했다.
- private Blob 이미지를 브라우저에서 볼 수 있도록 이미지 프록시 API를 추가했다.
- 확인된 주요 API:
  - `/api/promo-design-assets?runKey=promo-020`
  - `/api/promo-design-view?id=promo-020`
  - `/api/promo-design-image?id=promo-020`

### 4. n8n workflow 연결

- n8n `Persist UI Design Asset` 단계에서 생성 결과를 Vercel API로 POST 하도록 연결했다.
- n8n 결과에서 `imageUrl`, `designUrl`, `assetUrl`, `runKey`가 반환되는 흐름을 확인했다.
- 기존 n8n view webhook URL 대신 Vercel view URL을 사용하는 방향이 맞다.

### 5. Blob private store 대응

- poc_ax 프로젝트에서 사용 중인 Blob store를 promo-web-builder 프로젝트에도 연결하는 방향으로 진행했다.
- Blob store가 private access 기준이므로 직접 Blob URL을 노출하지 않고 `/api/promo-design-image` 프록시를 통해 이미지를 제공한다.
- `@vercel/blob` 구버전에서는 private access 옵션 오류가 발생해서 dependency를 최신 계열로 올렸다.

## 운영 설정 체크포인트

Vercel Production 환경에 아래 값이 필요하다.

- `DATABASE_URL` 또는 `NEON_DATABASE_URL`
- `BLOB_READ_WRITE_TOKEN`

주의:

- n8n의 저장 API URL은 현재 Production을 바라본다.
- 따라서 코드 변경 후에는 Preview 배포만으로는 n8n 테스트에 반영되지 않는다.
- n8n 테스트에 반영하려면 Production 재배포가 필요하거나, n8n URL을 Preview 도메인으로 임시 변경해야 한다.

## 오늘 발생했던 오류와 해결 기준

### 1. DB 테이블 없음

오류:

```text
relation "promo_design_runs" does not exist
```

원인:

- DB migration 미적용

해결:

- `004_promo_design_image_storage.sql` 적용

### 2. Blob public/private access 오류

오류 예시:

```text
Vercel Blob: Cannot use public access on a private store.
Vercel Blob: access must be "public"
```

원인:

- Blob store access 정책과 SDK 버전/업로드 옵션 불일치

해결:

- private store 기준으로 업로드하도록 구성
- `@vercel/blob` dependency를 `^2.5.0`으로 업데이트
- Blob 직접 노출 대신 이미지 프록시 API 사용

### 3. n8n view webhook에서 not found

오류:

```text
UI design not found
No generated UI design was stored for id: promo-017
```

원인:

- n8n static data나 실행 데이터에 이미지를 저장하는 기존 방식에 의존했기 때문
- 실행 데이터가 사라지거나 static data에 저장하지 않으면 view webhook에서 찾을 수 없음

해결:

- Vercel Blob + DB 저장 구조로 전환
- 보기 URL은 n8n webhook이 아니라 Vercel API 사용

## 다음 작업 제안

### 1. C섹션 디자인 목록 개선

- 현재는 생성 결과 저장과 조회가 되는 상태다.
- 다음 단계는 C섹션 디자인 목록을 DB/Blob 기준으로 다시 구성하는 것이 좋다.
- 방향:
  - 최근 생성 디자인을 타일 그리드로 표시
  - 썸네일 표시
  - 상태, 생성일, runKey, 보기 버튼 정리
  - 새로고침 후에도 DB에서 목록을 다시 불러오기

### 2. A섹션 Design MD 로딩 속도 개선

- 현재 Design MD 목록 로딩이 느린 원인은 목록 API가 raw markdown과 분석 데이터를 과하게 함께 내려주는 구조로 보인다.
- 개선 방향:
  - 목록 API는 요약 정보만 반환
  - 상세 raw markdown은 선택 시 lazy loading
  - A섹션 초기 렌더링 속도 개선

### 3. 외부 프롬프트 로딩 안정화

- UI Design Image Prompt는 외부 markdown으로 관리하는 방향이 맞다.
- 다만 Vercel API 또는 n8n HTTP Request 오류 시 workflow 전체가 실패하지 않도록 fallback 구조를 유지하는 것이 좋다.
- 다음 작업에서는 외부 prompt API와 n8n prompt loader를 다시 점검하면 된다.

### 4. n8n JSON 최신본 관리

- 로컬 기준 최신 workflow는 `n8n/promo-ui-design-image-generator.workflow.json`이다.
- 다운로드된 과거 n8n JSON에는 저장 단계가 빠져 있을 수 있으므로 재업로드 시 로컬 최신본 기준으로 확인해야 한다.
