# Handoff - 2026-06-30

## 오늘의 핵심 목표

선택된 Design MD와 B섹션 프로모션 요구사항 로그를 통합해 `Integrated Design Brief MD`를 만들고, 이 통합문서만 기준으로 UI 디자인 이미지 생성 요청이 이어지도록 구조를 정리했다.

## 주요 논의 정리

- `Integrated Design Brief MD`는 요약문이 아니라, downstream LLM이 이 문서만 보고 UI 디자인 생성 프롬프트를 만들 수 있는 독립 산출물이어야 한다.
- 기존 산출물은 `promo-integrated-design-brief-generation.md`의 의도와 다르게 느슨하게 생성될 수 있었다.
- 원인은 프롬프트 문서가 `type`, YAML frontmatter, sourceDocuments, height 정책을 강하게 고정하지 않았고, n8n 검증도 heading/길이 중심으로만 되어 있었기 때문이다.
- height 정책은 `1024x1536` 고정도 아니고 `3000px` 제한도 아니다.
- 최종 기준은 `1440px` desktop page width 기준, height는 콘텐츠에 따라 자유롭게 늘거나 줄며, Header부터 Footer까지 잘리지 않는 full-page Web UI design mockup이다.

## 반영된 내용

### 1. 통합문서 생성 프롬프트 외부화 및 강화

- 추가/수정:
  - `prompts/promo-integrated-design-brief-generation.md`
  - `api/prompts/promo-integrated-design-brief-generation.js`

- 반영 기준:
  - YAML frontmatter는 `---`로 시작/종료
  - `type: integrated_design_brief` 고정
  - `sourceDocuments`는 아래 3개로 고정
    - `Design MD`
    - `Promo Requirement Log MD`
    - `promo-integrated-design-brief-generation.md`
  - `pageWidth: 1440px`
  - `heightPolicy: auto height based on content; may be shorter or taller than 3000px`
  - `cropPolicy: never crop; include Header through Footer`
  - 모든 visible UI copy는 English only
  - `1024x1536`은 page height 기준으로 사용하지 않도록 수정

### 2. 이미지 생성 프롬프트 기준 보정

- 수정:
  - `prompts/promo-ui-design-image-generation.md`
  - `api/prompts/promo-ui-design-image-generation.js`

- 반영 기준:
  - `imageSize`는 모델이 지원하는 bitmap output size로만 해석
  - 실제 디자인은 flexible content-driven height의 full-page Web UI mockup으로 지시
  - vertical space가 부족하면 crop이 아니라 zoom out / section compression 방식으로 대응

## 남은 리스크

- 이미지 모델의 실제 bitmap size는 API가 허용하는 값으로 제한될 수 있다.
- 따라서 “height 자유”는 실제 픽셀 캔버스가 무제한이라는 뜻이 아니라, 프롬프트에서 full-page UI를 crop 없이 축소 표현하도록 유도하는 정책이다.
- n8n workflow JSON이 운영 환경에 import되지 않으면 로컬 변경만으로는 Cloud n8n 동작이 바뀌지 않는다.
- Vercel 배포 전에는 새 API와 Blob 저장 동작을 production에서 확인할 수 없다.
